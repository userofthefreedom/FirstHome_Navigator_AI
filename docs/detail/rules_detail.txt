FirstHome rules detail
작성 기준: 2026-06-24

이 문서는 FirstHome Navigator AI에서 "판단"이 들어가는 규칙을 한 곳에서 설명하기 위한 운영 문서다.
실제 규칙 코드는 backend/apps/rules/ 아래를 중심으로 두고, 각 app은 필요한 규칙을 import해 사용한다.
추천 점수, 자금 계산, 지역 매칭, 공고 분류, PDF 분석, 정책/금융/대출 후보 선정, AI 응답 안전장치가 이 문서의 대상이다.

중요한 전제
- 이 서비스는 소유형 공공분양 청약 준비를 돕는 참고 서비스다.
- 청약 신청 가능 여부, 당첨, 대출 승인, 정책 수급을 보장하지 않는다.
- 화면 계산보다 공식 공고문 원문, 금융기관 심사, 정책 기관 안내가 우선한다.
- fixture는 실제 데이터 부족을 보완하는 시연/안전망 데이터이며 실제 공고처럼 원문 링크를 제공하지 않는다.

1. 조건 입력값 저장과 단위

관련 파일
- frontend/src/pages/ProfileFormPage.vue
- frontend/src/stores/profileStore.js
- backend/apps/profiles/views.py
- backend/apps/profiles/models.py
- backend/apps/rules/funding.py
- backend/apps/rules/scoring.py

저장 흐름
- 비로그인 사용자는 Django session profile에 저장한다.
- 로그인 사용자는 UserProfile DB에 저장한다.
- 로그아웃하면 프론트 선택 상태와 임시 조건을 초기화한다.
- 로그인 사용자는 입력 조건, 선택 공고/옵션, AI 코치 캐시가 사용자 기준으로 이어진다.

단위
- 금액 입력 UI는 만원 단위로 받는다.
- API/DB/계산 내부 값은 원 단위다.
- 면적 입력 UI는 m2와 평 입력을 지원한다.
- API/DB/계산 내부 값은 m2 단위다.
- 평 변환은 1평 = 3.305785m2 기준이다.

계산에 직접 쓰지 않는 값
- name은 점수 계산에는 쓰지 않고 화면 문구와 AI 코치 호칭에 사용한다.

2. 자금 기본 규칙

관련 파일
- backend/apps/rules/funding.py
- backend/apps/funding/services/calculator.py
- backend/apps/rules/scoring.py

기본 비율
- DEFAULT_CONTRACT_RATE = 0.1
- DEFAULT_MIDDLE_PAYMENT_RATE = 0.6
- 공고/옵션에 실제 납부 일정이 있으면 실제 일정을 우선한다.
- 대표 공고값만 있을 때는 분양가 기준 계약금 10%, 중도금 60%, 잔금 30%로 기본 계산한다.

부채 반영
- debt_cash_reserve = debt * 0.3
- debt_monthly_reserve = ceil(debt / 60)
- available_cash = max(0, asset - debt_cash_reserve)
- effective_down_payment_cash = max(0, max_down_payment 또는 asset - debt_cash_reserve)
- effective_monthly_capacity = max(0, monthly_payment_capacity 또는 monthly_saving * job_status_factor - debt_monthly_reserve)

직업 상태 계수
- employed: 1.0
- self_employed: 0.95
- student: 0.9
- unemployed: 0.85
- 알 수 없는 값: 0.85
- 직업 상태는 최종 추천 점수 감점은 약하게, AI 코치 우선 확인 항목에는 강하게 반영한다.

자금 로드맵 표시 원칙
- "준비 가능 현금" 표시는 사용자가 입력한 보유 현금 흐름을 직관적으로 보여준다.
- 내부 자금 점수와 부족액 계산에는 부채 reserve를 반영할 수 있다.
- 부족액 표시는 기본적으로 계약금 - 현재 현금 관점이 사용자에게 가장 직관적이다.
- 융자금은 잔금처럼 입주 전 전액 납부하는 돈이 아니라 잔금 이후 대출 조건에 따라 상환할 금액으로 분리한다.

3. 청약 추천 점수

관련 파일
- backend/apps/recommendations/services/scoring.py
- backend/apps/recommendations/services/ranking.py
- backend/apps/rules/scoring.py

총점은 100점이다.
- eligibility: 35점
- funding: 25점
- location: 30점
- schedule: 10점

자격 점수 eligibility_score, 최대 35점
- 무주택이면 +8
- 청약통장 가입기간이 공고 요구 개월 이상이면 +7, 아니면 +3
- 공고 age_min <= 사용자 나이 <= age_max이면 +6, 아니면 +2
- notice.max_income이 있고 annual_income <= notice.max_income이면 +6
- notice.max_income이 없으면 정보 부족 보정값 +3
- notice.max_asset이 있고 asset <= notice.max_asset이면 +4
- notice.max_asset이 없으면 정보 부족 보정값 +2
- special_conditions가 공고 tag와 맞으면 +4
  - first_home + 생애최초
  - youth + 청년
- 특별조건이 맞지 않아도 기본 +1
- 최종값은 35점으로 clamp한다.

지역 점수 location_score, 최대 30점
- preferred_regions 중 하나가 공고 지역과 매칭되면 +24
- preferred_supply_types 중 하나가 공고 공급유형/주택유형과 매칭되면 +6
- 최종값은 30점으로 clamp한다.

일정 점수 schedule_score, 최대 10점
- 접수 마감일이 있고 마감까지 7일 이상 남으면 +3
- 접수 마감일이 있고 마감이 지나지 않았지만 7일 미만이면 +1
- 접수 마감일이 없거나 이미 지났으면 +0
- 계약일이 있고 계약일까지 30일 이상 남으면 +5
- 계약일이 있고 지나지 않았지만 30일 미만이면 +2
- 계약일이 없거나 이미 지났으면 +0
- 입주 예정값이 있고 2027-01 이후이면 +2
- 입주 예정값이 있고 2027-01 이전이면 +1
- 입주 예정값이 없으면 +0
- 최종값은 10점으로 clamp한다.

자금 점수 funding_score_from_plan, 최대 25점
- 계약금이 없거나 계산할 수 없으면 낮은 보정값 6점
- 준비율 = min(available_cash / down_payment, 1)
- 준비율 * 10점을 더한다.
- 부족액이 3,000만원 이하이면 +5
- 부족액이 5,000만원 이하이면 +3
- 그보다 크면 +1
- 월 준비 목표가 없으면 정보 부족 보정값 +4
- 월 저축 가능액이 없고 월 준비 목표가 있으면 +0
- 월 준비 목표가 effective_monthly_capacity 이하이면 +7
- 월 준비 목표가 effective_monthly_capacity * 1.5 이하이면 +4
- 그보다 크면 +1
- target_months >= 12이고 계약일까지 30일 이상이면 +3, 아니면 +1
- 날짜값은 누락/오류를 허용하며 최종값은 0~25점으로 clamp한다.

fixture 공고 감점
- backend/apps/recommendations/services/ranking.py
- fixture 공고는 실제 공고보다 낮게 보이도록 점수 보정을 한다.
- eligibility -10
- funding -4
- schedule -8
- location은 사용자가 고른 지역 일치성을 보여주는 축이라 감점하지 않는다.

4. 주택형 옵션 점수

관련 파일
- backend/apps/rules/scoring.py
- backend/apps/recommendations/services/ranking.py

옵션 우선순위
- general_supply: 4
- basic: 3
- pre_subscription: 2
- minus: 1
- 기타: 0

option_fit_score는 100점 기준이다.

면적 점수, 최대 35점
- exclusive_area_m2가 희망 최소/최대 범위 안이면 35점
- 범위를 벗어나면 초과/미달 비율만큼 감점한다.
- 감점식: max(0, 35 - gap_ratio * 100)
- 면적값이 없으면 0점

가격 점수, 최대 35점
- base_price가 희망 분양가 최소/최대 범위 안이면 35점
- 희망 최대보다 비싸면 max(0, 35 - gap_ratio * 100)
- 희망 최소보다 싸면 max(15, 35 - gap_ratio * 50)
- 가격값이 없으면 5점

계약금 점수
- 계약금이 있고 effective_down_payment_cash 안에 들어오면 +20
- 계약금 부족액을 target_months 동안 effective_monthly_capacity로 준비 가능하면 +14
- 계약금은 있지만 부담이 크면 +8
- 계약금이 없으면 +5

중도금/할부 일정 점수
- 월 필요액이 effective_monthly_capacity 이하이면 +10
- 1.5배 이하이면 +6
- 그보다 크면 +4
- 일정 데이터가 없으면 +5

입주 전 총액 점수
- move_in_cash_gap == 0이고 due_before_move_in > 0이면 +4
- gap이 있지만 6개월치 준비 여력 안이면 +2
- gap이 크면 -6

융자금 점수
- loan_amount > 0이고 loan_monthly_hint가 월 여력보다 크면 -4
- loan_amount > 0이지만 월 여력과 비교할 수 없거나 부담이 크지 않으면 -1
- due_before_move_in이 price 이하로 정상 범위면 +2
- 최종 옵션 점수는 0~100점으로 clamp한다.

대표 옵션 선정
- 공고별 옵션은 option_type_priority, option_fit_score, confidence 순으로 정렬한다.
- 사용자가 특정 옵션을 선택한 경우 그 선택을 유지한다.
- 공고만 선택된 경우 해당 공고 안의 대표 옵션을 사용한다.
- 아무 선택도 없으면 추천 1위 공고와 그 안의 대표 옵션을 사용한다.

5. 지역 규칙

관련 파일
- backend/apps/rules/regions.py

기본 원칙
- 대한민국 광역자치단체는 canonical label로 통일한다.
- 서울, 부산, 대구, 인천, 광주, 대전, 울산, 세종, 경기, 강원, 충북, 충남, 전북, 전남, 경북, 경남, 제주를 기본 권역으로 본다.
- 경기 북부/경기 남부는 공식 행정구역이 아니라 FirstHome 서비스 편의를 위한 하위 버킷이다.

경기 북부 키워드
- 고양, 김포, 남양주, 파주, 의정부, 양주시, 구리, 포천, 동두천, 가평, 연천

경기 남부 키워드
- 수원, 성남, 용인, 화성, 동탄, 평택, 오산, 안성, 이천, 여주, 광주, 하남, 과천, 안양, 군포, 의왕, 안산, 시흥, 광명, 부천, 양평

매칭 규칙
- preferred_region이 경기이면 경기도 전체를 허용한다.
- preferred_region이 경기 북부/남부이면 제목, 지역, 지구명, source_meta.supply_summary.district에서 하위 버킷 키워드를 찾는다.
- 하위 버킷 키워드가 있으면 그 버킷과 정확히 맞아야 한다.
- 공고 추천은 실제 공고 지구명이 별도 필드에 있는 경우가 많아, 하위 버킷 키워드가 없고 지역값이 단순 경기/경기도이면 보수적으로 매칭을 허용할 수 있다.
- 정책 매칭처럼 지역 텍스트가 짧거나 provider/name/target/benefit만 있는 데이터에서는 단순 경기/경기도가 경기 북부/남부에 자동 통과하지 않는다.
- 정책은 provider/name/target/benefit에서 지역을 먼저 추론한 뒤 전국 fallback을 적용한다.

6. 공고 분류 규칙

관련 파일
- backend/apps/rules/notice_classification.py
- backend/apps/notices/services/classifier.py

서비스 대상
- public_sale
- newlywed_public_sale
- private_participation_public_sale

강제 제외 키워드
- 행복주택, 국민임대, 영구임대, 매입임대, 장기전세, 든든전세, 전세, 신혼희망타운 임대
- 분양전환, 10년공공임대, 청년 공공주택
- 일반매각, 분양광고, 오피스텔, 상가, 임대

분류 우선순위
- 오피스텔/상가/토지는 서비스 범위 밖으로 제외한다.
- 신혼희망타운 + 공공분양이고 명시적 임대 신호가 없으면 newlywed_public_sale
- 민간참여 + 공공분양이고 명시적 임대 신호가 없으면 private_participation_public_sale
- 공공분양주택/공공분양/분양주택이고 명시적 임대 신호가 없으면 public_sale
- 위 소유형 공공분양 신호가 없고 강제 제외 키워드가 있으면 excluded 처리한다.
- 무순위/잔여세대/선착순 계열은 공공분양 여부 확인 필요 상태로 둔다.
- 그 외는 소유형 공공분양 여부 확인 필요 상태로 둔다.

명시적 임대 신호
- 행복주택, 국민임대, 영구임대, 매입임대, 장기전세, 분양전환, 공공임대는 임대 신호로 본다.
- 단, 공공분양 신호가 명확한 신혼희망타운/민간참여/공공분양 문구를 먼저 확인해 공공분양 공고가 단순 "임대" 단어 때문에 탈락하지 않도록 한다.

LH 공고 수집 보조 규칙
- 토지, 상가, 주차장, 주유소, 산업시설, 근린생활은 LH 공고 후보에서 제외한다.
- 주택, 임대, 분양, 행복, 국민, 영구, 매입, 전세, 공공 키워드를 주택 관련 후보 판별에 사용한다.
- 최종 서비스 노출은 위의 소유형 공공분양 분류를 다시 통과해야 한다.

7. PDF/공식 문서 분석 규칙

관련 파일
- backend/apps/rules/document_discovery.py
- backend/apps/rules/document_extraction.py
- backend/apps/rules/retrieval.py
- backend/apps/rules/confidence.py
- backend/apps/notice_docs/services/analysis.py

문서 후보 우선순위
- PDF이면 +50
- HWP/HWPX이면 +15
- 입주자모집공고문/입주자 모집공고문/모집공고문이면 +80
- 입주자모집/모집공고/공고문이면 +40
- 팸플릿, 브로슈어, 위임장, 동의서, 견본주택, 안내문, 서식, 동호, 배치도, 평면도, 발코니, 옵션, cad, dwg는 -45

금액 섹션 판별
- 주택가격, 계약금, 중도금, 잔금, 공급금액, 분양가격 중 3개 이상 있으면 가격 섹션으로 본다.

옵션 추출 유효성
- 전용면적은 20m2 이상 120m2 이하만 유효하다.
- 분양가는 1억원 이상 20억원 이하만 유효하다.
- 최대 추출 옵션 수는 50개다.

융자금 판별 키워드
- 융자금, 대출금, 주택도시기금

옵션 유형 판별
- 사전청약 또는 계약금 5% 문구가 있으면 pre_subscription
- 일반 당첨자/특별공급/일반공급 또는 계약금 10% 문구가 있으면 general_supply
- 마이너스 문구가 있으면 minus
- 그 외는 basic

공식 확인 체크리스트
- 무주택 기준 확인
- 소득·자산 기준 확인
- 청약통장 요건 확인
- 지역 우선공급 확인
- 각 항목은 키워드 수, 선호 문구, 페이지 번호, 기준/판정 문구를 기준으로 근거 페이지와 신뢰도를 계산한다.
